# Adapters

Agent QA uses adapters to interact with external systems. This allows you to customize how tests communicate with your infrastructure.

## Overview

| Adapter | Purpose | Built-in Implementation |
|---------|---------|------------------------|
| `DatabaseAdapter` | Entity queries | Drizzle ORM |
| `AgentAdapter` | AI agent communication | HTTP client |
| `VectorStoreAdapter` | Vector store queries | Milvus |
| `TracesProvider` | Trace collection | Grafana Tempo |

## When to Use Custom Adapters

Use a custom adapter when:

- Your database uses a different ORM than Drizzle
- Your agent uses a non-HTTP protocol (WebSocket, gRPC)
- You use a different vector store than Milvus
- You use a different tracing backend (LangFuse, Jaeger, Datadog)
